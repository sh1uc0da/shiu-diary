SHIU DIARY · BACKEND SETUP

詩雨國日記・串接教學

把展示版接上真正的 Google 後端，讓所有粉絲跨裝置共用同一本交換日記。全程免費、不用自己的伺服器。預計 20–30 分鐘。

☂ 展示模式

config.js 兩個值留空即可。資料存在這台瀏覽器、登入是模擬的，適合預覽與試玩。什麼都不用設定。

✦ 正式模式

填好後端網址與 Google 金鑰後，資料改存 Google 試算表、跨使用者共用，並使用真正的 Google 登入。

1

建立一份 Google 試算表

到 sheets.new 建一份新的空白試算表，命名成「詩雨國日記資料庫」。工作表內容稍後由程式自動建立，現在不用手動加任何欄位。

2

打開 Apps Script，貼上後端程式碼

在剛剛的試算表上方選單點：擴充功能 → Apps Script。把編輯器裡原本的 function myFunction(){} 全部刪掉，改貼上專案裡 Code.gs 的完整內容，按上方 💾 儲存。

設定後台密碼：在 Code.gs 最上方把 const ADMIN_PASSWORD = 'rain2026' 改成你自己的密碼——這是 CODA 進後台用的，存在後端、不會外洩給前端。
3

執行一次 setup 建立工作表

在 Apps Script 上方的函式下拉選單選 setup，按 ▶ 執行。第一次會跳出授權視窗，選你的 Google 帳號 →「進階」→「前往（不安全）」→ 允許。完成後試算表會自動長出 Diaries、Comments、Likes 三個工作表。

4

部署成 Web App，取得後端網址

Apps Script 右上角 部署 → 新增部署作業，齒輪選 網頁應用程式，設定：

  • 說明：隨意（例如 v1）
  • 執行身分：我
  • 誰可以存取：任何人

按部署、授權，複製產生的網址（結尾是 /exec）。

日後修改 Code.gs 後，要回到「部署 → 管理部署作業 → ✏️ 編輯 → 版本選『新版本』」重新部署，改動才會生效。
5

建立 Google 登入金鑰

到 Google Cloud Console：

  • 建立一個新專案（或用現有的）。
  • 左側 API 和服務 → OAuth 同意畫面：使用者類型選「外部」，填好 App 名稱、你的信箱即可（測試階段把自己加進測試使用者）。
  • API 和服務 → 憑證 → 建立憑證 → OAuth 用戶端 ID，應用程式類型選 網頁應用程式。
  • 在 已授權的 JavaScript 來源填入你網站的網域，例如 https://你的名字.github.io（之後上線的網址）。
  • 建立後複製 用戶端 ID（結尾 .apps.googleusercontent.com）。
注意：Google 登入只能在「真正的網域」上運作，不能用 file:// 直接打開檔案。所以這一步要先想好網站要放哪（見步驟 7）。
6

把兩個值填進 config.js

打開專案裡的 config.js，把步驟 4 和 5 拿到的兩個值貼進去：

// config.js
window.SHIYU_CONFIG = {
  BACKEND_URL: "https://script.google.com/macros/s/AKfy.../exec",
  GOOGLE_CLIENT_ID: "1234567890-abcdefg.apps.googleusercontent.com",
  DEMO_ADMIN_PASSWORD: "rain2026",
};

填好後重新整理，前端就會自動切到正式模式（真資料 ＋ 真登入）。

7

把網站上線

把整個專案資料夾（所有 .html、.css、.js、.jsx）放到任一免費靜態空間，例如 GitHub Pages、Netlify、Cloudflare Pages。上線後的網址，記得回到步驟 5 加進「已授權的 JavaScript 來源」。

隱私與安全

  • 匿名（維持現狀）：時間軸與留言只顯示暱稱與標籤，其他粉絲看不到真實姓名／信箱。
  • 私密日記：後端只在驗證後台密碼後才回傳私密內容，外人就算拿到網址也讀不到。
  • 不要把試算表設成公開分享——資料一律只透過 Apps Script 進出。
  • 後台密碼存在 Code.gs，不在前端，不會被使用者看到。
